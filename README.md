# USSD-Based Student Result Checker and Secure Class Attendance Management System

**Institution:** CKT-UTAS  
**Short Code:** `*419#`  
**Stack:** Node.js · Express · MySQL · Africa's Talking USSD API

---

## Features

| Module | Features |
|---|---|
| **USSD – Results** | Student authenticates by Index No. + PIN; sees all published grades + CGPA |
| **USSD – Attendance** | 5-layer fraud prevention (code, phone, time window, enrollment, duplicate check) |
| **USSD – Exam Info** | Check exam date / time / venue by course code |
| **USSD – Profile** | View name, index number, programme, level, department |
| **Admin Dashboard** | Students, Courses, Results, Exams, Attendance sessions, Activity logs |
| **USSD Simulator** | Built-in browser simulator – no phone needed for local testing |
| **Security** | bcrypt PIN hashing, JWT, rate limiting, SQL-injection-safe parameterized queries, PIN lockout after 5 failures |

## Quick Start

### Prerequisites
- Node.js ≥ 18
- MySQL 8 (XAMPP, WAMP, or native)

### 1. Clone / unzip and install

```bash
cd ussd-result-attendance-system
npm install
```

### 2. Configure environment

```bash
cp .env.example .env
```

Open `.env` and set your MySQL credentials:

```
DB_HOST=127.0.0.1
DB_PORT=3306
DB_USER=root
DB_PASSWORD=          # leave blank for XAMPP default
DB_NAME=ussd_result_attendance
JWT_SECRET=any_long_random_string_here
```

### 3. Initialize the database

```bash
node database/init.js
```

This creates the database, all tables, and seeds sample data.

**Default login:**

- Lecturer: `jmensah`  same password

**Sample student:**

### 4. Start the server

```bash
npm start       # production
npm run dev     # with nodemon (auto-restart on file change)
```

Open **http://localhost:3000** for the admin dashboard.

## Connecting Africa's Talking (Production)

1. Create a free account at [africastalking.com](https://africastalking.com)
2. In Sandbox → USSD → create service code `*419#`
3. Set the callback URL to `https://your-domain.com/ussd`
   - For local testing, expose your port with [ngrok](https://ngrok.com): `ngrok http 3000`
4. Set `AT_USERNAME` and `AT_API_KEY` in your `.env`

## API Endpoints

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/ussd` | none | Africa's Talking USSD callback |
| POST | `/api/auth/login` | none | Admin/Lecturer login → JWT |
| GET | `/api/auth/me` | JWT | Current user |
| GET/POST/PUT/DELETE | `/api/students` | JWT | Student CRUD |
| GET/POST/PUT/DELETE | `/api/courses` | JWT | Course CRUD |
| POST | `/api/courses/enroll` | JWT | Enroll student in course |
| GET/POST/PUT/DELETE | `/api/results` | JWT | Results CRUD |
| POST | `/api/results/bulk` | JWT | Bulk upload results |
| POST | `/api/results/publish` | JWT | Publish results |
| GET/POST/PUT/DELETE | `/api/exams` | JWT | Exam schedule CRUD |
| POST | `/api/attendance/sessions` | JWT | Create session + generate code |
| PUT | `/api/attendance/sessions/:id/close` | JWT | Close session |
| GET | `/api/attendance/sessions` | JWT | List sessions |
| GET | `/api/attendance/sessions/:id/report` | JWT | Attendance report |

| GET | `/api/dashboard/stats` | JWT | Dashboard statistics |

## Attendance Security Layers

1. **Phone Number Verification** – only the student's registered number can submit
2. **Active Session Check** – lecturer must have created a session for that course
3. **Attendance Code Validation** – student enters the code generated by the lecturer
4. **Time Window Restriction** – code is valid for a configurable window (default 20 min)
5. **Enrollment Check** – student must be enrolled in the course
6. **Duplicate Prevention** – unique constraint prevents double attendance (DB + app level)
